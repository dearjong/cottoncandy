## 1. 개요

본 문서는 ADMarket Admin 시스템의 업무 흐름을 **MVP 관점(플랫폼 지장 없는 운영 흐름)** 으로 정리한 v0.3 초안입니다.

각 흐름은 아래 항목을 포함합니다.
- 설명
- 현재 구현 경로(라우트/화면)
- 구현 상태(구현됨/부분 구현)

**민감정보 열람 원칙(중요)**

- 대화/계약/제안/정산/증빙 등 **민감정보는 개별 회원/기업/프로젝트 화면에서 상시 열람하지 않고**,
  `보안/감사 > 보안자료`(` /admin/security/messages `)에서 **필터 선택 → 검색**으로만 열람한다.
  (분쟁/감사 등 사유가 있을 때만 접근하는 구조)

## 2. 프로젝트 운영 업무 흐름 (MVP)

### 2.1 프로젝트 승인 / 중단·취소 / 전체 운영

| 흐름 | 설명 | 현재 구현 경로(예시) | 상태 | 앞으로 구현할 것 |
| --- | --- | --- | --- | --- |
| 프로젝트 승인(공개) | 승인 대기 프로젝트 검토 후 승인/반려 | `/admin/pending-approval`, `ProjectManagement` 승인/반려 다이얼로그 | 구현됨 | — |
| 프로젝트 등록/수정 검토 | 수정/공개 여부 등 운영 정책에 따른 검토 | `/admin/pending-approval` + 프로젝트 상세 | 부분 구현 | 1) 상세에서 **승인/반려** 버튼 제공<br/>2) **반려 사유** 입력(필수일 경우 저장 불가)<br/>3) 처리 후 목록에서 **상태/라벨**이 즉시 갱신<br/>4) 상세 상단에 **처리 결과(승인됨/반려됨 + 사유)**가 보임 |
| 중단/취소 요청 처리 | 중단/취소 요청 승인/거절 | `/admin/stop-cancel` + `ProjectManagement` 중단/취소 팝업 | 구현됨 | — |
| 프로젝트 상세 운영 뷰 | 개요/참여/일정/계약·산출물/정산/리뷰 탭으로 운영 | `/admin/project-detail/:id`, `ProjectManagement` + 상세 탭 | 부분 구현 | 1) 각 탭에 **운영자 액션 버튼이 있으면**: 클릭 → 처리 결과가 즉시 반영<br/>2) 각 탭에 **조회만**이면: 입력/수정 UI가 아예 안 보이도록 고정<br/>3) “지금 상태에서 가능한 탭/액션”만 노출(불가능한 건 비활성/숨김) |
| 참여/제안/매칭 운영(절차별) | 참여 기업/제안/OT·PT 등 절차별 운영 | `/admin/workflow/*` | 부분 구현 | 1) 각 단계에서 운영자 액션을 명시(예: **확정/거절/다음단계 이동** 중 무엇인지)<br/>2) 처리 후 **현재 단계/상태가 화면에 표시**<br/>3) 단계 화면에서 관련 문서/상세로 **클릭 한 번에 이동** 가능 |

## 6. 시스템·보안·감사 업무 흐름 (MVP)

| 흐름 | 설명 | 현재 구현 경로(예시) | 상태 | 앞으로 구현할 것 |
| --- | --- | --- | --- | --- |
| 관리자 계정 관리 | 관리자 계정 CRUD 및 역할/권한 구성(기본) | `/admin/settings` + `AdminSettings` | 부분 구현 | 1) 관리자 목록에서 **추가/수정/비활성** 가능<br/>2) 역할(예: Super/Admin/CS 등)을 선택 가능<br/>3) 역할 변경 후 접근 가능한 메뉴가 실제로 달라짐 |
| 플랫폼 설정 | 유지보수 모드 및 운영 플래그 설정 | `/admin/settings/platform` | 부분 구현 | 1) 유지보수 모드 토글 시 사용자 화면에 **점검 상태**가 반영<br/>2) 설정 화면에 적용 범위(전체/일부)가 표시 |
| 사용자 로그 보기 | 사용자 활동 로그 조회 | `/admin/settings/logs`, `/admin/activity-logs` | 부분 구현 | 1) 검색(키워드) + 필터(기간/유형) 제공<br/>2) 권한 없는 운영자는 접근 불가/숨김 |
| 보안자료 접근/관리 | 민감정보 열람(증빙 검색) 운영 화면 | `/admin/security/messages` | 부분 구현 | 1) 자료 유형 필터(대화/계약/제안/산출물/정산/증빙 등) 선택<br/>2) 키워드/프로젝트ID 등으로 검색<br/>3) 열람 전 **사유 입력**(필수) + 열람 기록이 남음(정책)<br/>4) 민감정보는 마스킹되고, 권한에 따라 추가 열람 제한 |
