## 1. 알림 개요

- 알림은 **정보 전달 목적**이며, 프로젝트 상태 판단의 기준은 상태값(state)이 우선한다.
- 알림 누락이 발생해도 상태값은 유지된다.
- 알림은 3개 채널로 전달된다: **플랫폼 내 알림 센터 / 이메일 / 외부 채널(카카오톡 등, 향후)**

---

## 8. 알림 메시지 포맷 (권장)

```
{이벤트 제목}
---
[프로젝트명]: {project_name}
{상세 내용}

> [바로가기] → {deep_link_url}
```

**변수 목록**

| 변수 | 설명 |
| --- | --- |
| `{project_name}` | 프로젝트명 |
| `{actor_name}` | 액션을 수행한 사용자/기업명 |
| `{reason}` | 반려·보완 사유 |
| `{datetime}` | 일정 일시 |
| `{item}` | 정산 항목명 (계약금/중도금/잔금) |
| `{version}` | 시안 버전 번호 |
| `{deep_link_url}` | 해당 화면 딥링크 |
