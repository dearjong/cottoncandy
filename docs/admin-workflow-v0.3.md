## 1. 개요

본 문서는 ADMarket Admin 시스템의 **업무 흐름도(v0.2)** 를 기반으로,  
현재 cottoncandy 레포에서 **이미 구현된 흐름**과 **향후 구현이 필요한 흐름**을 함께 정리한 v0.3 초안입니다.

**표기 규칙**

- **상태**
  - `구현됨` : 현재 레포에 화면/플로우가 존재 (목업/가짜 데이터여도 OK)
  - `부분 구현` : 화면은 있으나, 문서에 정의된 액션/자동화/정책이 일부만 있음
  - `미구현` : UI/플로우가 아직 없음
- **Phase**
  - `MVP` : 지금 라인에서 우선 구현할 범위
  - `P2` : 운영 안정화 이후(스마트 전 단계)
  - `P3` : 스마트/AI·자동화 수준에서 구현

> 참고: 일괄 처리(배치)는 플랫폼 지장 없이도 운영이 가능하므로, v0.3(MVP)에는 미구현/옵션 항목을 제외합니다.

> 참고: 스마트/자동화(자동 승인·이상탐지·예측·자동리포트)는 플랫폼 지장 없이도 운영이 가능하므로, v0.3(MVP)에서는 제외합니다.
