## 1. 개요

ADMarket은 광고주, 대행사, 제작사를 연결하는 광고 제작 매칭 플랫폼입니다. 본 기능 정의서는 플랫폼을 구성하는 모든 기능에 대한 명확한 정의와 요구사항을 제시합니다. 기획, 디자인, 개발, 운영 모든 조직은 본 문서에 정의된 기능 명세를 따라야 합니다.

## 2. 공통 기능 (Common Features)

### 2.1. 헤더 (Header)

헤더는 모든 페이지의 상단에 위치하며, 사용자가 주요 기능에 빠르게 접근할 수 있도록 설계되었습니다.

| 요구ID | 기능명 | 설명 | 대상 사용자 | 승인 기준 |
|---|---|---|---|---|
| HDR-001 | 로고 표시 및 메인 이동 | TVCF ADMarket 로고를 표시하며, 클릭 시 메인 페이지로 이동합니다. | 전체 | 로고가 정상적으로 노출되고, 클릭 시 메인 페이지로 이동해야 합니다. |
| HDR-002 | 메인 메뉴 표시 | '광고제작 의뢰하기', '대행사·제작사 찾기', '프로젝트 공고' 등 주요 서비스로 이동하는 GNB(Global Navigation Bar) 메뉴를 표시합니다. | 전체 | 메뉴가 정상적으로 노출되고, 각 메뉴 클릭 시 해당 페이지로 정확히 이동해야 합니다. |
| HDR-003 | 알림(Notification) 표시 | 새로운 활동(메시지, 프로젝트 상태 변경, 참여 신청 등)이 있을 경우, 아이콘에 배지(badge)를 표시하고 클릭 시 알림 팝업을 엽니다. | 회원 | 새로운 알림이 있을 때 배지가 표시되고, 클릭 시 알림 목록 팝업이 정상적으로 작동해야 합니다. |
| HDR-004 | 메시지함 표시 | 새로운 쪽지가 있을 경우, 아이콘에 배지(badge)를 표시하고 클릭 시 메시지함 페이지로 이동합니다. | 회원 | 새로운 쪽지가 있을 때 배지가 표시되고, 클릭 시 메시지함 페이지로 정상적으로 이동해야 합니다. |
| HDR-005 | 사용자 프로필 표시 | 로그인한 사용자의 기업명, 담당자 이름 등 프로필 정보를 표시합니다. 클릭 시 '설정', '내 정보', '로그아웃' 등이 포함된 드롭다운 메뉴를 제공합니다. | 회원 | 프로필 정보가 정확히 표시되고, 드롭다운 메뉴가 정상적으로 작동해야 합니다. |
| HDR-006 | 로그인/회원가입 버튼 | 비로그인 상태의 사용자에게 로그인 및 회원가입 페이지로 이동하는 버튼을 표시합니다. | 전체 | 비로그인 상태에서만 버튼이 노출되어야 합니다. |
| HDR-007 | 모바일 메뉴 (햄버거 버튼) | 모바일 화면에서 메뉴 및 유틸리티를 접힐 수 있는 햄버거 버튼을 제공합니다. | 전체 | 모바일 화면에서 햄버거 버튼이 정상적으로 노출되고, 클릭 시 메뉴가 접혔다 펼쳐져야 합니다. |

| FTR-003 | 주요 메뉴 링크 | 플랫폼 소개, 1:1 문의, FAQ, 공지사항 등 주요 정보 페이지로 이동하는 링크를 제공합니다. | 전체 | 모든 링크가 정상적으로 해당 페이지로 이동해야 합니다. |
| FTR-004 | 정책/약관 링크 | 이용약관, 개인정보처리방침, 서비스 정책 등 법적 문서 링크를 제공합니다. | 전체 | 모든 정책 문서 링크가 정상적으로 해당 페이지로 이동해야 합니다. |
| FTR-005 | 카피라이트 표시 | © TVCF ADMarket 표시를 노출합니다. | 전체 | 카피라이트가 정상적으로 표시되어야 합니다. |
| FTR-006 | 모바일 반응형 푸터 | 모바일 화면에서 푸터 메뉴가 접혀 있다가 클릭 시 펼쳐지는 형태로 제공합니다. | 전체 | 모바일에서 푸터가 정상적으로 접혔다 펼쳐져야 합니다. |

## 3. 핵심 기능 (Core Features)

### 3.1. 회원 및 계정 관리

회원 및 계정 관리 기능은 사용자의 가입, 로그인, 프로필 관리, 탈퇴 등을 담당합니다.

| 요구ID | 기능명 | 설명 | 대상 사용자 | 승인 기준 |
|---|---|---|---|---|
| MEB-001 | 통합 로그인 | TVCF 기존 회원 계정을 사용하여 ADMarket에 별도 가입 없이 로그인할 수 있습니다. SSO(Single Sign-On) 방식으로 구현됩니다. | 전체 | TVCF 계정으로 로그인 및 서비스 이용이 정상적으로 이루어져야 합니다. |
| MEB-002 | 신규 회원가입 | 이메일 인증, 휴대폰 인증, 기본 정보(이름, 별명, 비밀번호), 직업 정보 등록을 통해 신규 회원을 가입 처리합니다. | 전체 | 모든 인증 절차와 정보 입력이 정상적으로 처리되고, 가입이 완료되어야 합니다. |
| MYI-001 | 개인정보 관리 | 사용자는 자신의 개인정보(이름, 이메일, 휴대폰 등)를 등록하고 수정할 수 있습니다. | 회원 | 개인정보 등록 및 수정이 정상적으로 이루어져야 합니다. |
| MYI-005 | 알림 설정 | 사용자는 받고 싶은 알림의 종류와 채널(이메일, 카카오톡 등)을 설정할 수 있습니다. | 회원 | 알림 설정이 정상적으로 저장되고, 설정에 따라 알림이 전달되어야 합니다. |
| MYI-006 | 회원 탈퇴 | 사용자가 서비스를 탈퇴할 수 있는 기능을 제공하며, TVCF 계정과 연동되어 동시 탈퇴 처리됩니다. 진행 중인 프로젝트가 있는 경우 탈퇴가 제한될 수 있습니다. | 회원 | 탈퇴 시 관련 데이터 처리 정책에 따라 정보가 삭제되고, TVCF 계정과의 연동 탈퇴가 정상적으로 이루어져야 합니다. |

### 3.2. 프로젝트 의뢰 및 등록 (Create Project)

프로젝트 의뢰 및 등록 기능은 광고주나 대행사가 새로운 프로젝트를 플랫폼에 등록하는 과정을 담당합니다.

| 요구ID | 기능명 | 설명 | 대상 사용자 | 승인 기준 |
|---|---|---|---|---|
| CPJ-001 | 광고제작 의뢰 진입 | 사용자는 광고 제작을 의뢰하기 위한 프로세스를 시작할 수 있습니다. 비회원도 초기 단계까지 진입 가능합니다. | 전체 | 로그인 여부와 관계없이 프로젝트 의뢰 초기 단계 진입이 가능해야 합니다. |
| CPJ-002 | 임시저장 불러오기 | 이전에 작성하던 프로젝트 내용이 있는 경우, 이를 불러와 이어서 작성할 수 있습니다. 시스템은 자동으로 임시저장 여부를 감지하고 팝업으로 안내합니다. | 회원 | 임시저장된 내용이 있을 경우, 팝업을 통해 사용자에게 알리고, 선택 시 내용이 정상적으로 복원되어야 합니다. |
| CPJ-003 | 파트너 찾는 방법 선택 | 사용자는 공개 공고, 비공개 직접의뢰, 컨설턴트 의뢰 중 하나를 선택할 수 있습니다. | 전체 | 각 의뢰 방식 선택이 정상적으로 작동해야 합니다. |
| CPJ-004 | 파트너 유형 선택 | 사용자는 원하는 파트너 유형(대행사, 제작사 등), 의뢰 내용, 일정 대응 능력 등을 선택합니다. | 전체 | 선택 옵션이 정상적으로 작동해야 합니다. |
| CPJ-005 | 제목 입력 후 로그인 안내 | 프로젝트 제목을 입력한 후 다음 버튼을 클릭하면, 비회원 사용자에게 로그인 안내 팝업이 표시됩니다. 로그인 후 작성한 제목이 유지되어야 합니다. | 전체 | 로그인 팝업이 정상적으로 표시되고, 로그인 후 제목이 유지되어야 합니다. |
| CPJ-006 | 프로젝트 제목 등록 | 프로젝트의 명확한 제목을 등록합니다. (예: "2026년 상반기 신제품 광고 제작") | 회원 | 제목이 정상적으로 저장되어야 합니다. |
| CPJ-007 | 광고 목적 선택 | 광고의 주요 목적(브랜드 인지, 제품 판매, 이벤트 홍보 등)을 선택합니다. | 회원 | 선택 옵션이 정상적으로 작동해야 합니다. |
| CPJ-008 | 제작 기법 선택 | 원하는 제작 기법(실사, 애니메이션, 모션그래픽 등)을 선택합니다. | 회원 | 선택 옵션이 정상적으로 작동해야 합니다. |
| CPJ-009 | 제작 형태·노출 매체 선택 | 제작 형태(TVC, 디지털, 라디오 등)와 노출 매체를 선택합니다. | 회원 | 선택 옵션이 정상적으로 작동해야 합니다. |
| CPJ-010 | 주요 타겟 고객 선택 | 광고의 주요 타겟 고객층(연령, 성별, 관심사 등)을 선택합니다. | 회원 | 선택 옵션이 정상적으로 작동해야 합니다. |
| CPJ-011 | 예산 구간 선택/확정금액 입력 | 프로젝트 규모별 예산 구간을 선택하거나, 확정된 금액을 직접 입력할 수 있습니다. | 회원 | 예산 입력이 정상적으로 저장되어야 합니다. |
| CPJ-012 | 대금 지급 비율 선택 | 계약금, 중도금, 잔금의 지급 비율을 선택합니다. (예: 30% / 40% / 30%) | 회원 | 지급 비율 설정이 정상적으로 저장되어야 합니다. |
| CPJ-013 | 일정 등록 | 프로젝트의 주요 일정(접수, 자료 제출 마감, 결과 발표, 납품, 온에어)을 등록합니다. | 회원 | 일정이 정상적으로 저장되고, 캘린더 형식으로 표시되어야 합니다. |
| CPJ-014 | 제품 정보 등록 | 광고 대상 제품의 정보(제품명, 카테고리, 특징 등)를 등록합니다. | 회원 | 제품 정보가 정상적으로 저장되어야 합니다. |
| CPJ-015 | 담당자 정보 등록 | 프로젝트 담당자의 이름, 연락처, 이메일 등을 등록합니다. | 회원 | 담당자 정보가 정상적으로 저장되어야 합니다. |
| CPJ-016 | 경쟁사 수행 기업 선택/제외 | 경쟁사가 수행한 기업을 선택하여 참여 대상에서 제외할 수 있습니다. | 회원 | 제외 기업 설정이 정상적으로 작동해야 합니다. |
| CPJ-017 | 참여 기업 조건 선택 | 참여 기업의 상세 조건(회사 규모, 경력, 수상 경력 등)을 선택합니다. | 회원 | 조건 선택이 정상적으로 작동해야 합니다. |
| CPJ-018 | 제출 자료 선택·파일 업로드 | 참여사가 제출해야 할 자료(제안서, 시안 등)를 선택하고, 별도 양식이 있을 경우 파일을 업로드합니다. | 회원 | 파일 업로드가 정상적으로 작동해야 합니다. |

| CPJ-020 | 프로젝트 승인 신청 | 모든 정보를 입력한 후, 플랫폼 운영자에게 프로젝트를 검토 및 승인해달라고 요청합니다. 승인 신청 시 운영자에게 알림이 전달됩니다. | 회원 | 승인 요청이 정상적으로 시스템에 등록되고, 운영자에게 알림이 전달되어야 합니다. |

### 3.3. 파트너 찾기 (Find Partners)

파트너 찾기 기능은 광고주나 의뢰자가 원하는 조건에 맞는 대행사나 제작사를 검색하고 연락할 수 있도록 합니다.

## 6. 기능 적용 원칙

### 6.1. 권한 관리
모든 기능은 사용자의 역할과 상태에 따라 접근 권한이 제어됩니다. 비회원, 개인회원, 기업회원 등 각 사용자 유형별로 접근 가능한 기능이 명확하게 정의되어 있습니다.

### 6.2. 데이터 일관성
모든 기능에서 입력되는 데이터는 일관된 형식과 검증 기준을 따릅니다. 데이터 중복 입력을 방지하고, 자동 연동 기능을 활용합니다.

### 6.3. 사용자 경험
모든 기능은 사용자의 편의성을 최우선으로 설계되었습니다. 명확한 안내, 직관적인 인터페이스, 빠른 응답 속도를 보장합니다.

### 6.4. 보안
민감한 정보(계약서, 정산 정보 등)는 암호화되어 저장되며, 접근 권한이 엄격하게 제어됩니다.
