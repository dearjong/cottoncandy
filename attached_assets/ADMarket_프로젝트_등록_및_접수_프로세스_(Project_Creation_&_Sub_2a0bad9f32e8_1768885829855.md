# ADMarket 프로젝트 등록 및 접수 프로세스 (Project Creation & Submission)

### **1️⃣ 개요 (Overview)**

| 항목 | 내용 |
| --- | --- |
| **기능명** | 프로젝트 등록 및 접수 |
| **설명** | **의뢰인(광고주·대행사·컨설턴트)** 이 프로젝트를 등록하고, 조건에 맞는 **맞춤형 파트너(대행사/제작사)** 를 찾는 과정 |
| **등록 주체** | 광고주 / 대행사 / 컨설턴트 |
| **프로젝트 유형** | 공개 프로젝트 / 컨설팅 의뢰형 / 비공개 직접 의뢰 |
| **승인 정책** | - 공개 프로젝트: 운영자 승인 필요- 컨설팅 의뢰형: 자동 승인- 비공개 직접 의뢰: 승인 불필요 |
| **OT/PT 단계** | 선택적(Optional) – 모든 유형에서 설정 가능 |
| **자동화 요소** | 상태 전이, 접수 마감 자동 처리, 알림 발송, 로그 저장 |
| **연동 모듈** | `project`, `submission`, `contract`, `schedule`, `notification` |

### **2️⃣ 프로젝트 주요 단계 (Main Flow – 프로젝트 메인 상태 기준)**

| 대분류 | 하위 상태 | 상태 코드 | 설명 |
| --- | --- | --- | --- |
| **등록 (REGISTER)** | 임시저장 | `DRAFT` | 의뢰인이 프로젝트를 작성 중이며 임시로 저장된 상태 |
|  | 승인요청 | `REQUESTED` | 의뢰인이 프로젝트 승인 요청을 보낸 상태 |
|  | 승인완료 | `APPROVED` | 운영자 승인 또는 자동 승인 완료 |
|  | 접수중 | `PROPOSAL_OPEN` | 참여자(대행사/제작사)가 제안 또는 참여 신청을 할 수 있는 상태 |
|  | 접수마감 | `PROPOSAL_CLOSED` | 제안 및 참여 신청이 마감된 상태 |
| **매칭/선정 (MATCHING)** | OT 예정 | `OT_SCHEDULED` | 오리엔테이션 일정이 예정된 상태 |
|  | OT 완료 | `OT_COMPLETED` | 오리엔테이션이 완료된 상태 |
|  | 제안서 제출 대기 | `PROPOSAL_SUBMIT` | 참여자가 제안서를 준비 중인 상태 |
|  | 제안서 제출 완료 | `PROPOSAL_SUBMITTED` | 참여자가 제안서를 제출 완료한 상태 |
|  | PT 예정 | `PT_SCHEDULED` | 프리젠테이션 일정이 예정된 상태 |
|  | PT 완료 | `PT_COMPLETED` | 프리젠테이션이 완료된 상태 |
|  | 미선정 | `NO_SELECTION` | 참여자가 선정되지 않은 상태 |
|  | 선정완료 | `SELECTED` | 참여자가 최종 선정된 상태 |
| **계약 (CONTRACT)** | 계약완료 | `CONTRACT` | 계약 체결이 완료된 상태 |
| **제작 (PRODUCTION)** | 촬영 중 | `SHOOTING` | 광고 촬영이 진행 중인 상태 |
|  | 후반 작업 | `EDITING` | 편집 및 후반 작업이 진행 중 |
|  | 시안 제출 | `DRAFT_SUBMITTED` | 시안(초안)이 제출된 상태 |
|  | 최종본 컨펌 완료 | `FINAL_APPROVED` | 최종 결과물이 승인 완료된 상태 |
| **제작완료 (PRODUCTION_COMPLETE)** | — | `PRODUCTION_COMPLETE` | 제작이 완전히 종료된 상태 |
| **온에어 (ONAIR_STARTED)** | — | `ONAIR_STARTED` | 광고 온에어가 시작된 상태 |
| **사후관리 (AFTER_SERVICE)** | — | `AFTER_SERVICE` | **제작리뷰 등록 및 사후 관리 단계** |
| **완료 (COMPLETE)** | — | `COMPLETE` | 프로젝트가 완전히 종료된 상태 |
| **중단 (STOPPED)** | 관리자 확인중 | `ADMIN_CHECKING` | 중단 요청 접수 및 검토 중 |
|  | 관리자 승인됨 | `ADMIN_CONFIRMED` | 프로젝트 중단이 승인된 상태 |
| **취소 (CANCELLED)** | 관리자 확인중 | `ADMIN_CHECKING` | 취소 요청 접수 및 검토 중 |
|  | 관리자 승인됨 | `ADMIN_CONFIRMED` | 프로젝트 취소가 승인된 상태 |

### **3️⃣ 프로젝트 유형별 주요 분기 (Type-Specific Differences)**

| 구분 | 공개 프로젝트 | 컨설팅 의뢰형 | 비공개 직접 의뢰 |
| --- | --- | --- | --- |
| **등록 주체** | 광고주 / 대행사 | 컨설턴트 | 광고주 / 대행사 |
| **운영자 승인** | 필요 (등록 후 운영자 검수) | 자동 승인 (컨설턴트 등록 시) | 승인 불필요 |
| **접수 방식** | **공개 지원** — 여러 참여자가 자유롭게 지원하거나 제안서를 제출 | **컨설턴트 추천** — 컨설턴트가 조건에 맞는 참여자를 직접 추천하거나, **공고를 대신 등록해 참여자를 모집**하며, 필요 시 **비공개 1:1 직접 의뢰**도 수행 가능 | **1:1 직접 의뢰** — 의뢰인이 특정 참여자에게만 개별 제안 |
| **PT/OT 진행** | 선택 (의뢰인 설정 가능) | 선택 (컨설턴트가 대신 진행 가능) | 선택 (필요 시 개별 조율) |
| **계약 주체** | 의뢰인 ↔ 참여자 | 의뢰인 ↔ 컨설턴트 ↔ 참여자 | 의뢰인 ↔ 지정 참여자 |
| **공개 범위** | 전체 공개 (ADMarket 전체 회원 노출) | 비공개 중심 (단, 컨설턴트가 필요 시 공개형으로 전환 가능) | 완전 비공개 (초대된 참여자만 접근 가능) |

---

💡 **요약:**

컨설팅 의뢰형은

> “컨설턴트가 프로젝트를 대행하며, 상황에 따라
> 
> 
> 공개 모집 / 직접 추천 / 비공개 1:1 의뢰 중 가장 적합한 방식을 선택할 수 있는 구조.”
> 

### **4️⃣ 주요 참여자 (Actors)**

| 구분 | 설명 | 주요 액션 포인트 (Action Point) |
| --- | --- | --- |
| **의뢰인 (광고주·대행사·컨설턴트)** | 프로젝트를 등록하고, 유형(공개/컨설팅/비공개)에 따라 진행 방식을 결정하는 주체 | 프로젝트 등록 → 승인 요청(공개형) → 참여자 검토 → 계약 확정 |
| **참여자 (대행사·제작사)** | 프로젝트에 참여하기 위해 제안서를 제출하거나 초대에 응하는 주체 | 참여 신청(공개형) → 제안서 제출 → PT/OT 참석 → 계약 체결 → 제작 진행 |
| **컨설턴트 (Consultant)** | 의뢰인의 요청을 받아 프로젝트 등록 및 진행을 대행하는 전문가. 공개 모집, 직접 추천, 1:1 의뢰 모두 수행 가능 | 의뢰 수락 → 프로젝트 등록 → 모집 또는 추천 진행 → 계약 지원 및 관리 |
| **운영자 (Admin)** | 플랫폼 정책, 승인, 검수, 통계, 제재를 담당하는 내부 관리자 | 프로젝트 승인·반려 → 기업 인증 관리 → 컨설턴트 이관·모니터링 |
| **시스템 (System)** | 상태 전이, 알림, 일정, 로그, 접수 마감 등을 자동 관리 | 이벤트 발생 시 자동 처리 (상태 변경, 알림 발송, 로그 기록 등) |

---

💡 **정리**

- **의뢰인**은 등록 및 의사결정 주체
- **참여자**는 제안 및 실행 주체
- **컨설턴트**는 의뢰인을 대신하여 위임 수행
- **운영자**는 승인·검수 및 관리 주체
- **시스템**은 자동화 로직을 담당
