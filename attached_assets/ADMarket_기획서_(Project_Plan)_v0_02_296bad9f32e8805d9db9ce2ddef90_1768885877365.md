## 1장. 프로젝트 개요 (Overview)

---

### **1️⃣ 프로젝트명**

**ADMarket (애드마켓)**

ADMarket은 20년간 광고 DB를 쌓아온 TVCF가 만드는 [광고주&대행사&제작사] 매칭 및 표준 프로젝트 관리 플랫폼입니다.
이 플랫폼은 비딩, AI 추천, 전문가 추천 등 다양한 기능을 통해 광고주와 제작사가 손쉽게 협업 할 수 있는 환경을 제공합니다.
프로젝트 의뢰부터 계약, 제작, 리뷰, 정산까지 모든 과정을 한 곳에서 관리할 수 있어, 광고주와 제작사는 시간과 비용을 절감하면서도 고품질의 광고를 제작할 수 있습니다.

---

### **2️⃣ 한 줄 정의**

> “대한민국 광고 산업을 20년간 기록해온 TVCF, 이제 데이터 기반 [광고주&대행사&제작사]를 연결하는 플랫폼으로 진화합니다.”
> 

---

### **3️⃣ 프로젝트 목적**

ADMarket은 광고주·대행사·제작사가

**데이터로 검증된 파트너를 손쉽게 찾을 수 있도록 연결해주는 광고 제작 매칭 플랫폼**이다.

- 광고주는 **신뢰할 수 있는 제작사**를 쉽게 찾고,
- 제작사는 **브랜드급 프로젝트**를 통해 포트폴리오를 강화하며,
- TVCF는 **20년 광고 DB 자산을 수익 모델로 전환**한다.

> 💡 핵심 문장:
> 
> 
> “인맥이 아니라, 데이터로 연결되는 광고 생태계.”
>

## 2장. 프로젝트 배경 (Background & Needs)

---

### **1️⃣ 광고 산업의 오래된 문제**

광고 제작 시장은 겉보기엔 화려하지만, 실제로는 **비공식적 연결과 감(感)에 의존**해 돌아간다.

대형 광고일수록 인맥 중심으로 거래가 이루어지고,

신뢰는 개인의 평판에 기대며,

정작 **“좋은 제작사가 제대로 선택 받지 못하는 구조”**가 유지되고 있다.

> “누가 잘 하느냐 보다, 누가 아느냐가 중요한 시장.”
> 

이 구조는 결국 광고주·제작사 모두에게 손해를 남긴다.

| 문제 영역 | 현상 |
| --- | --- |
| **광고주 입장** | 프로젝트를 맡길 회사를 찾을 때 ‘검증된 기준’이 없다. |
| **제작사 입장** | 좋은 작품을 만들어도 브랜드에게 닿을 통로가 없다. |
| **산업 전체** | 수많은 데이터가 있지만, 신뢰를 만들어내는 구조가 없다. |

---

### **2️⃣ 기존 플랫폼이 놓친 핵심**

지난 몇 년간 수많은 광고·외주 플랫폼이 생겼다.

하지만 그들은 ‘가격’과 ‘속도’에 초점을 맞췄고,

**“신뢰”를 시스템 안에 담는 데는 실패했다.**

| 플랫폼 | 특징 | 구조적 한계 |
| --- | --- | --- |
| **크몽 / 숨고** | 프리랜서 중심, 견적 경쟁 구조 | 대형 광고주에게는 품질 불신 |
| **위시켓 / 이랜서** | 개발·디자인 프로젝트 중개 | 광고 제작 특유의 협업 구조 반영 불가 |
| **마담** | 광고 전문 매칭  **상담형 운영 구조
대행사/제작사 비용부담** | 상담 중심 → **자동화 부재**, 처리 속도 느림, 
데이터 축적 어려움 |
| **TVCF (기존)** | 20년 광고 데이터 보유 | 데이터는 있으나, 연결 기능이 없음 |

광고 제작 매칭 플랫폼은 이미 존재하지만, **전문성을 기준으로 자동화된 매칭을 구현한 플랫폼은 없었다.**

ADMarket은 TVCF의 축적된 데이터와 매칭 자동화를 결합해 

**빠르고, 정확하고, 효율적인 파트너 연결 구조**를 만든다.

---

### **3️⃣ TVCF의 20년 자산**

TVCF는 지난 20년간

대한민국 광고 산업의 거의 모든 기록을 쌓아왔다.

| 항목 | 보유 자산 |
| --- | --- |
| **광고 데이터** | 업종·브랜드·캠페인별 약 30만 편 |
| **제작사 정보** | 수천 개 기업, 담당자·감독·스태프 DB |
| **수상 내역** | 국내외 광고제 수상 데이터 (Cannes, D&AD 등) |
| **작품 정보** | 감독·PD·에이전시별 필모그래피 구조화 |

이건 단순한 기록이 아니다.

이건 “**광고 산업의 신뢰 데이터베이스**”다.

> 지금까지 TVCF는 ‘기록의 플랫폼’이었다면,
> 
> 
> 이제 ADMarket은 그 기록을 ‘계약의 플랫폼’으로 바꾼다.
> 

---

### **4️⃣ ADMarket이 해결하는 핵심 Pain Point**

| 대상 | 기존 문제 | ADMarket의 해결 방식 |
| --- | --- | --- |
| **광고주** | 신뢰할 제작사를 찾기 어려움 | 조건 기반 검색 + 검증된 DB 매칭 |
| **제작사 / 대행사** | 포트폴리오 노출 기회 부족 | TVCF 데이터 기반 노출 + 리뷰 연동 |
| **운영자(TVCF)** | 보유 DB의 수익화 부재 | 매칭·계약·리뷰 기반 신뢰 생태계 구축 |

> 💡 “ADMarket은 광고 산업의 신뢰를 다시 설계하는 시스템이다.”
> 

---

### **5️⃣ 지금 ‘이 타이밍’에 필요한 이유**

- **① 시장의 피로감이 한계에 도달함**
    
    → 광고주는 더 이상 감(感)에 의존하고 싶지 않다.
    
    → 제작사는 ‘평판이 아닌 결과’로 평가 받길 원한다.
    
- **② 기술적으로 가능한 시점**
    
    → 자동화·AI 추천·데이터 분석 기술이 현실화 됐다.
    
    → 인맥 대신 데이터가 연결을 만드는 구조가 준비됐다.
    
- **③ TVCF의 성장 정체 구간**
    
    → 보유 DB를 ‘보는 자산’에서 ‘움직이는 자산’으로 전환해야 할 시점이다.
    

> 🎯 결론:
> 
> 
> ADMarket은 ‘시장 불만’과 ‘TVCF의 자산’이 만나는 지점이다.
> 
> 이 타이밍을 놓치면, 20년의 데이터는 다시 아카이브로 남게 된다.
> 

---

### **6️⃣ 한 문장 요약**

> “ADMarket은 광고 산업이 미뤄왔던 문제를,
TVCF만이 해결할 수 있는 방식으로 푸는 프로젝트다.”

## 3장. 서비스 비전 및 핵심 가치 (Vision & Core Values)

---

### 1) 비전 (Vision)

> “광고를 만든 사람들의 기록으로, 다음 광고의 파트너를 정확히 찾는다.”
> 
> 
> TVCF의 20년 광고 DB를 **신뢰 데이터**로 전환해, 광고주·대행사·제작사가 **데이터 기반으로 연결**되는 표준 생태계를 만든다.
> 

---

### 2) 미션 (Mission)

- **광고주**: 감(感)이 아니라 **검증된 데이터**로 파트너 선택.
- **제작사/대행사**: **공식 포트폴리오·수상·리뷰**로 정당하게 선택받음.
- **TVCF**: “기록 플랫폼”을 넘어 **“계약을 만드는 데이터 플랫폼”**으로 진화.

### **3️⃣ 핵심 가치 (Core Values)**

---

| 핵심 가치 | 의미 | ADMarket에서의 구현 방식 |

### **4️⃣ 운영 철학 (Operating Philosophy)**

ADMarket은 **소기업형 구조**를 전제로 설계됐다.

많은 인력 없이도, 시스템이 스스로 일하게 만든다.

| 철학 | 설명 |
| --- | --- |
| **운영비 0원 구조** | 컨설턴트가 상담하지 않아도 자동으로 매칭 가능 |
| **단계적 개발 (MVP)** | 등록 → 매칭 → 계약 → 리뷰의 4단계로 핵심 기능만 우선 구현 |
| **단순한 진입 (무료 정책)** | 초기에는 완전 무료로, 진입 장벽 최소화 |
| **명확한 권한 구조** | 기업회원 [대표관리자/부관리자/일반직원]
개인회원 권한 구분으로 내부 혼란 방지 |
| **데이터 중심 성장** | 서비스의 가치는 매칭 수가 아니라 축적된 신뢰 데이터 양으로 판단 |

> ⚙️ 한 줄 요약:
> 
> 
> “사람이 아니라 정책이, 정책이 아니라 시스템이 운영하게 한다.”
>

## 4장. 서비스 구조 및 사용자 계정

---

### **1️⃣ 기본 원칙 (Account)**

- 로그인은 **TVCF 통합 계정 기반(DB 공유)** 이며, 한 번 가입하면 양쪽 플랫폼 모두 이용 가능하다.
- 한개의 계정 ID에 1개의 휴대전화 번호와 여러개의 이메일 등록이 가능하다.
- 단, ADMarket의 **활동·권한·구성원 데이터는 독립 관리**된다.
- ADMarket은 **하나의 계정으로 개인계정 / 기업 기업계정으로** 변경이 가능하다.
- 기업 퇴사 등의 이벤트에 따라 소속 상태가 자동 전환된다.
- 회원은 **프로젝트를 의뢰(발주)** 하거나 **제작사로 참여**할 수 있다.

> 💡 핵심 철학:
> 
> 
> “한 계정으로 광고를 맡기고, 또 다른 프로젝트에 참여할 수도 있는 유연한 구조.”

## 5장. 프로젝트 정책

---

### 🎯 **1. 개요**

ADMarket의 모든 프로젝트는

> 등록 → ADMarket운영자 승인 → 참여기업 모집 → OT → PT → 계약 → 제작 → 정산 → 제작리뷰 → 종료의 **표준 절차(Standard Flow)** 로 진행된다.
> 

각 단계의 실행 여부와 결과는 시스템에 자동 기록되어

**매칭·추천의 신뢰 데이터로 축적**된다.

> 💡 핵심 문장
> 
> 
> “모든 프로젝트는 기록되고, 그 기록이 신뢰가 된다.”
> 

---

### 🧭 **2. 프로젝트 등록 방식 (3가지 진입 경로)**

| 유형 | 설명 | 주요 특징 |
| --- | --- | --- |
| **① 공개 프로젝트 등록** | 광고주 또는 대행사가 프로젝트를 공개 등록 → 조건이 맞는 제작사가 자유 참여 | 오픈형 매칭 / 참여 확대 / 산업 DB 축적 |
| **② 비공개 직접 의뢰** | 광고주가 특정 조건(예산·분야·경험)에 맞는 제작사에 1:1 또는 소수 초청 | 맞춤형 / 보안 강화 / 빠른 진행 |
| **③ 컨설턴트에게 맡길게요** | ADMarket 공식 컨설턴트가 프로젝트 방향 설정 및 대행/제작사 선정 대행 | 유료(20만원~) / 전문가 추천 / 관리 대행 |

> ⚙️ 운영 원칙
> 
> - 모든 등록 방식은 동일한 Flow를 따른다.
> - 공개 프로젝트 → **운영자 승인 필수**
> - 비공개·컨설팅 의뢰 → **승인 없이 즉시 진행 가능**
> - **컨설팅 서비스 유료**

### 📑 **7. 계약 (Contract)**

- 계약은 광고주 ↔ 제작사 간 **직접 체결**
- ADMarket은 **계약서 업로드·보관·정산 관리 기능만 제공**
- 법적 책임은 계약 당사자에게 있으며,
    
    계약서·견적서·NDA 등은 프로젝트 단위로 자동 저장
    

> 📄 Note: “ADMarket은 계약을 대신하지 않는다.
> 
> 
> 다만 계약이 ‘정확히 기록되도록’ 보조한다.”
>

### 🎬 **8. 제작 (Production)**

- 제작사는 일정(간트차트) 등록 및 산출물 업로드
- 광고주는 시안 검수 및 수정 요청 가능(메세지로 발신)
- 승인 시 자동으로 정산 단계로 전환
- 진행 과정은 프로젝트 히스토리로 축적

> 💡 “모든 제작 기록(일정지연 등)이 차후 평가와 추천의 데이터가 된다.”
>

| **데이터 활용** | 정산 완료 시 자동 기록 → 신뢰 지표 반영 |

### 🔚 **11. 전체 프로젝트 흐름 (Standard Flow)**

```
등록 → 관리자 승인 → 참여기업 모집 → OT → PT → 계약 → 제작 → 정산 → 제작리뷰 → 종료
```

| 단계 | 설명 | 담당 주체 | 선택 여부 |
| --- | --- | --- | --- |
| **등록** | 프로젝트 정보 입력 및 제출 | 광고주 / 대행사 / 개인 | 필수 |
| **관리자 승인** | 공개·컨설팅 프로젝트 검수 | Admin | 조건부 |

## 8장. 회원 및 기업 계정 정책 (User & Company Account Policy)

---

### 🧭 **1. 개요**

ADMarket의 회원은 하나의 계정으로 관리되며,

**개인정보에 기업 정보가 없는 경우 ‘개인회원(Personal)’으로**,

**기업 정보가 있고 기업 관리자의 활동 승인을 받은 경우 ‘기업회원(Company)’으로**

자동 분류된다.

모든 회원은 기본적으로 **TVCF 통합 계정 DB**를 기반으로 관리되며,

한 번의 로그인으로 TVCF와 ADMarket을 모두 이용할 수 있다.

기업 회원은 **대표관리자 / 부관리자 / 일반직원**으로 역할이 구분되며,

다만, ADMarket은 대표 또는 관리자 승인 하에만 활동(프로젝트 등록, 참여 등)이 가능하다.

> 💡 핵심 원칙:
> 
> 
> “모든 사용자는 하나의 계정으로,
> 
> 역할과 권한에 따라 다르게 작동한다.”

# 📘 9장. 기업 프로필 및 역량 데이터 정책

---

## 🎯 **목적**

- **TVCF의 신뢰 DB를 기반으로** 기업의 전문성, 경험, 신뢰 정보를 구조화한다.
- **기업 프로필 = 회사소개서 자동생성 구조**로, 모든 개인/기업은 자동으로 자기소개서를 보유하며 이를 프로젝트 제안 시 복사·수정해 활용할 수 있다.
- 기업의 전문분야와 최소 제작비를 함께 관리하여 **정확한 매칭, 검색, 추천**을 구현한다.

---

## 🧩 **데이터 구조**

### **1️⃣ TVCF DB → ADMarket 자동 생성**

- ADMarket 기업 회원 생성 시, TVCF DB에서 자동으로 아래 항목을 가져와 **“기본 회사소개서(자동생성)”** 생성
    - 기업 기본정보 (회사명, 주소, 사이트, 설립연도 등)
    - 대표 광고주 / 최근 작업
    - 대표 스태프
    - 특화산업 / 경험분야
    - 전문매체
    - 수상내역

> 📌 이 기본 소개서는 ADMarket 기업 프로필의 초안(기본형) 으로 자동 저장된다.
> 
> 
> 관리자는 “복사본 만들기”를 통해 프로젝트별, 클라이언트별 버전을 자유롭게 수정·활용 가능.
> 

---

### **2️⃣ ADMarket 내 독립 관리**

- 자동 생성된 회사소개서는 기업 계정 내에서 **복사, 수정, 확장 가능**
- 수정 및 추가 가능 항목:
    - **대행사:** 광고 목적별 전문분야
    - **제작사:** 제작 기법별 전문분야
    - **공통 항목:** 대표 클라이언트 / 전문매체 / 수상내역 / 핵심 스태프 / 최소 제작비 등
- 변경 내용은 관리자 검토 후 노출 반영

> 💡 “TVCF의 기업 회원은 ADMarket에서 자동으로 회사소개서를 갖고 태어나며,
> 
> 
> 그 소개서를 복사해 맞춤 제안서로 활용한다.”
>

- 복잡한 승인 절차 대신 **투명한 인증 배지 중심 구조**를 채택한다.

# 📘 10장. 프로젝트 등록 및 승인 정책

### (Project Registration & Approval Policy)

---

### 🎯 **1. 목적**

ADMarket의 프로젝트 등록 프로세스는

광고 제작 의뢰를 **투명하고 구조적으로 관리**하기 위한 핵심 단계이다.

프로젝트 등록은 단순한 공고 게시가 아니라,

**기업 인증 → 관리자 승인 → 참여 모집 → 계약까지 이어지는 일련의 신뢰 절차**로 설계되어 있다.

> 💡 핵심 문장:
> 
> 
> “누구나 공고를 올릴 수 있지만,
> 
> 신뢰받는 프로젝트만 공개된다.”
> 

---

### 🧩 **2. 등록 방식 (Request Style)**

프로젝트는 광고주 또는 담당자가 목적과 진행 방식에 따라

3가지 형태 중 하나로 등록할 수 있다.

| 등록 유형 | 설명 | 예시 |
| --- | --- | --- |
| **① 공개 프로젝트 (Public)** | 참여기업 모집을 위한 공개 공고형 | “브랜드 캠페인 신규 제작사 모집” |
| **② 비공개 직접의뢰 (Private Direct)** | 특정 기업에만 비공개로 제안 | “마음에 드는 회사에 비공개 제안” |
| **③ 컨설턴트 의뢰 (Consultant-assisted)** | 전문가가 대행하여 제작사 구성 및 관리 | “20만원부터 맞춤 컨설팅 의뢰” |

---

### 🪜 **3. 전체 프로세스 (Flow Overview)**

```
등록 → 관리자 승인 → 참여기업 모집 → OT → PT → 계약 → 제작 → 정산 → 리뷰 → 종료
```

| 단계 | 설명 |
| --- | --- |
| **등록** | 광고주 또는 담당자가 프로젝트 의뢰 등록 |
| **관리자 승인** | 운영자가 검토 후 승인 (공개형만 해당) |
| **참여기업 모집** | 제작사/대행사 지원 접수 (참여신청 마감일 기준) |
| **OT / PT** | 일정 등록, 기업별 발표 진행 (선택적) |
| **계약** | 선정 기업과 계약서 등록 및 승인 |
| **제작** | 일정에 따라 제작 진행 |
| **정산 / 리뷰 / 종료** | 완료 후 정산 및 쌍방 리뷰 작성 |

---

### 🧾 **4. 플랫폼 운영자 승인 정책 (Platform Operator Approval)**

| 항목 | 공개 프로젝트 | 비공개 의뢰 | 컨설턴트 의뢰 |
| --- | --- | --- | --- |
| **승인 필요 여부** | ✅ 승인 필수 | ❌ 승인 불필요 | ✅ 컨설턴트 검수 후 승인 |
| **검토 항목** | - 프로젝트 진정성 여부 검토
(기업 정보, 예산 합리성, 항목 누락 여부)

# 📘 11장. 매칭 및 추천 정책

### (Matching & Recommendation Policy)

---

### 🎯 **1. 목적**

ADMarket의 매칭 시스템은

광고주의 요구 조건과 기업의 전문 역량을 자동 분석하여

**“가장 잘 맞는 파트너를 빠르고 신뢰 있게 추천”**하는 기능이다.

기존 플랫폼이 ‘가격’이나 ‘키워드’ 중심이었다면,

ADMarket은 **TVCF의 신뢰 DB와 실적 데이터**를 활용해

**경험·전문성·예산 적합도·활동 이력**을 기반으로 한다.

> 💡 핵심 문장:
> 
> 
> “AI가 단순히 연결하지 않고,
> 
> ‘누가 이 일을 가장 잘할 수 있는가’를 찾아낸다.”
>

### 🧩 **2. 매칭 개요**

### 기본 구조

```
[광고주 입력조건] ↔ [기업 역량 데이터] → [매칭 점수 계산] → [추천·필터·알림]
```

| 구분 | 입력 데이터 | 예시 |
| --- | --- | --- |
| **광고주 조건** | 광고 목적, 산업군, 매체, 제작기법, 예산, 일정 | “브랜딩 목적 / F&B / TV+SNS / 예산 8천만 원” |
| **기업 데이터** | 전문분야, 산업경험, 매체경험, 최소제작비, 수상, 참여이력 | “3D+실사 / F&B 다수 / TV·SNS 전문 / 최소비 5천만 원” |

### ⚙️ **3. 매칭 알고리즘 개요 (Matching Score Model)**

| 평가 항목 | 가중치(%) | 설명 |
| --- | --- | --- |
| **전문분야 일치도** | 35% | 광고 목적(대행사) 또는 제작기법(제작사) 일치율 |
| **산업군 일치도** | 20% | 광고주 입력 산업군과 기업 특화 산업 비교 |
| **매체 적합도** | 10% | 주요 노출 매체(SNS, TV, OTT 등) 일치율 |
| **예산 적정성** | 15% | 기업 최소 제작비 ≤ 프로젝트 예산 |
| **수상 / 인증 가산점** | 10% | TVCF 인증기업, 수상이력, 포트폴리오 완성도 |
| **활동이력 / 평판** | 10% | 프로젝트 성공률, 리뷰 점수, 응답률 등 |

> 🧮 총점 = (일치도×가중치) 합계
> 
> 
> 85점 이상: 우수매칭 / 70점 이상: 적합 / 50점 미만: 제한 표시
>

| **AI 추천 (Smart Suggest)** | 행동데이터·선호기업 기반 학습 | “맞춤공고” 및 “AI 제안기업” |

### 🔁 **4. 순환형 구조 (Positive Loop)**

```
플랫폼 내 거래 증가
       ↓
Verified Deal 누적
       ↓
신뢰도·데이터 자산 상승
       ↓
AI 매칭·리포트·인증 서비스 수익 증가
       ↓
플랫폼 가치 상승 → 신규 이용자 유입
```

> 📈 “거래는 한 번이지만, 데이터는 자산이 된다.”
> 

---

### 🎯 **5. 플랫폼 내 거래 유도**

ADMarket은 직거래를 막지 않는다.

하지만, 플랫폼 내에서 거래하면 **눈에 보이는 이득**이 생긴다.

| 항목 | 혜택 | 설명 |
| --- | --- | --- |
| **Verified Deal 배지** | 거래 완료 기업 인증 | 신뢰·노출 강화 |
| **포트폴리오 자동등록** | TVCF DB 연동 | 홍보효과 |
| **리워드 포인트** | 계약/리뷰 완료 시 적립 | 반복 이용 유도 |
| **자동 리포트 생성** | 계약·정산·리뷰 데이터 기반 | 회계·대외 활용 |
| **매칭 가산점** | Verified Deal 가중치 반영 | 신규 매칭 유리 |

> 💬 “직접 해도 되지만, 플랫폼 안에서 하면 더 많이 남는다.”
>

### **6. 데이터 중심 수익 모델 (Revised)**

- **1단계:** **TVCF 광고 DB + ADMarket 거래 데이터 통합**
    
    → 광고 제작사, 대행사, 광고주 간 매칭·계약·성과 데이터가 축적되는 구조
    
    → 산업별, 캠페인별, 제작규모별 실데이터 확보
    
- **2단계:** **데이터 시각화 및 산업 리포트 제공**
    
    → 축적된 데이터를 기반으로 광고 트렌드, 산업군별 제작비 분석, 수상/성과 통계 리포트 발행
    
    → 브랜드/대행사/제작사별 벤치마킹 서비스
    
- **3단계:** **AI 기반 예측 및 매칭 모델 고도화 (구독형 판매)**
    
    → 광고주 행동 패턴 + 프로젝트 성과 데이터 기반으로
    
    “AI 추천 매칭” 정교화 및 SaaS 형태로 외부 기관/대행사 판매
    
- **4단계:** **데이터 상용화 및 API 기반 확장 (TVCF Data Cloud)**
    
    → KPI 연동, 산업별 리포트 API, 외부 파트너사와 데이터 연계
    
    → 대한민국 광고 산업 표준 데이터 플랫폼으로 확장

# 📘 **부록 A. 용어 정의 (Glossary)**

---

| 용어 | 정의 |
| --- | --- |
| **광고주 (Advertiser)** | 광고를 제작·집행하기 위해 프로젝트를 의뢰하는 주체. 기업, 브랜드 담당자, 혹은 대행사일 수도 있다. |
| **대행사 (Agency)** | 광고 기획과 제작을 총괄하는 조직으로, 광고주의 목표를 바탕으로 제작사 및 스태프를 구성·관리한다. |
| **제작사 (Production Company)** | 영상, 그래픽, 음향 등 광고 제작의 실제 실행을 담당하는 기업 또는 팀. |
| **OT (Orientation)** | 프로젝트 착수 전, 참여 기업과 광고주가 처음 만나 방향성과 일정 등을 공유하는 오리엔테이션 미팅. |
| **PT (Presentation)** | 광고주가 참여 기업 중 후보를 선정하기 위한 발표 단계. 제안서, 콘티, 아이디어 등을 심사한다. |
| **컨설턴트 매칭 
(Consultant Matching)** | 광고주가 직접 의뢰하지 않고, ADMarket의 전문 컨설턴트가 조건에 맞는 제작사/대행사를 선별·추천하는 유료 서비스. |
| **리젝션 Fee (Rejection Fee)** | PT 또는 제안서 경쟁 이후 탈락한 기업에 지급되는 보상금. 공정 경쟁을 유도하기 위한 제도. |
| **Verified Deal (공식 거래 인증)** | 플랫폼 내에서 계약, 정산, 리뷰까지 완료된 거래에 부여되는 인증 마크. 신뢰도를 나타내며 프로필 노출에 반영된다. |
| **기업 인증 (Company Verification)** | 사업자등록증, 대표자 정보 등을 확인해 기업 신뢰를 보증하는 절차. 인증 기업은 별도 마크로 표시된다. |
| **AI 추천 (AI Recommendation)** | 광고주 입력 조건과 기업 활동 데이터를 분석해 가장 적합한 파트너를 자동 추천하는 기능. |
| **KPI (Key Performance Indicator)** | 광고 캠페인의 성과를 정량적으로 측정하기 위한 핵심 지표. (예: 조회수, 전환율, 매출 상승률 등) |
| **TVCF 인증 (TVCF Certification)** | ADMarket과 통합된 TVCF 플랫폼의 기업 신뢰 인증. 실제 광고 제작 이력, 포트폴리오, 수상 기록 등을 기반으로 부여된다. |
| **프로젝트 (Project)** | 광고 제작을 위한 단위 업무. 광고주/대행사가 등록하고, 대행사/제작사가 참여하여 제작, 계약, 정산까지 완료한다. |
| **프로젝트 담당자 (Project Owner)** | 프로젝트 등록·관리 책임자. 개인회원이거나 기업 소속 직원일 수 있다. |
| **참여신청 
(Participation Request)** | 제작사·대행사가 프로젝트에 참여하기 위해 제출하는 신청. 승인 시 PT 또는 제안서 제출 단계로 이어진다. |
| **제안서 (Proposal)** | 제작사 또는 대행사가 광고주의 요구 조건에 맞춰 제출하는 제안 문서. 아이디어, 콘티, 견적 등이 포함된다. |
| **정산 (Settlement)** | 계약 조건에 따라 대금이 지급되는 과정. 계약금·중도금·잔금 구조로 구분되며, 플랫폼은 증빙 보관 역할을 담당한다. |
| **리뷰 (Review)** | 프로젝트 완료 후 광고주와 제작사가 서로에 대한 평가를 남기는 단계. 평점과 코멘트가 기업 신뢰도에 반영된다. |
| **직거래 (External Deal / Bypass)** | ADMarket 외부에서 진행되는 거래. 금지되지 않지만 플랫폼 내 거래 시 더 많은 혜택과 인증이 주어진다. |
| **Verified Report (거래 리포트)** | 플랫폼 내에서 거래를 완료한 기업에게 자동 생성되는 거래 요약 문서. 계약서, 일정, 정산, 리뷰 정보를 포함한다. |
| **컨설팅 범위 (Consulting Scope)** | 컨설턴트가 제공하는 서비스 수준. 예산·제작 규모·산업군에 따라 단계별로 다르다. |
| **프로젝트 승인 (Project Approval)** | 광고주가 등록한 프로젝트를 플랫폼 운영자가 검토 후 공개하는 절차. 
비공개 직접의뢰는 승인 제외. |
| **포트폴리오 (Portfolio)** | 제작사·대행사가 진행한 과거 광고 사례 모음. Verified Deal 완료 시 자동으로 갱신된다. |
