## 1. 개요

본 기능 정책서는 ADMarket 플랫폼의 주요 기능에 대한 운영 정책을 정의합니다. 모든 사용자와 운영자는 본 문서에 기술된 정책을 준수해야 합니다. 이 정책서는 기능정의서의 보완 문서로서, 각 기능이 어떤 원칙과 기준에 따라 운영되어야 하는지를 명시합니다.

---

## 2. 계정 및 회원 정책

### 2.1. 회원 가입 및 로그인 정책

**통합 계정 관리**

ADMarket의 계정은 TVCF 통합 계정을 기반으로 합니다. TVCF 회원은 별도의 가입 절차 없이 ADMarket을 이용할 수 있으며, 신규 사용자는 이메일 및 휴대폰 인증을 통해 가입할 수 있습니다. 한 개의 계정 ID에 여러 개의 이메일을 등록할 수 있으나, 휴대폰 번호는 1개만 등록 가능합니다.

**로그인 정책**

사용자는 TVCF 통합 계정으로 로그인하며, 이를 통해 양 플랫폼의 서비스를 모두 이용할 수 있습니다. 로그인 세션은 30일 동안 유지되며, 보안상 이유로 필요시 재인증을 요구할 수 있습니다.

**회원 탈퇴 정책**

회원이 탈퇴를 신청할 경우, 다음 조건을 확인합니다:
- 진행 중인 프로젝트가 없어야 합니다.
- 담당 업무가 모두 이관되어야 합니다.
- 미정산 금액이 없어야 합니다.

탈퇴 시 TVCF 계정과 ADMarket 계정이 동시에 탈퇴 처리되며, 관련 데이터는 법정 보존 기간에 따라 관리됩니다.

### 2.2. 계정 유형 정책

**개인 계정**

개인 계정은 기업에 소속되지 않은 사용자가 사용합니다. 개인 계정 사용자는 다음 활동을 수행할 수 있습니다:
- 개인 광고주로서 프로젝트 의뢰
- 프리랜서 또는 제작사로서 프로젝트 참여
- 기업 생성 및 대표관리자 역할 수행

**기업 계정**

기업 계정은 기업 정보를 등록하고, 해당 기업의 대표관리자 또는 부관리자로부터 활동을 승인받은 계정입니다. 기업 계정 사용자는 다음 활동을 수행할 수 있습니다:
- 기업 자격으로 프로젝트 의뢰 또는 참여
- 기업 프로필 및 포트폴리오 관리
- 구성원 관리 (대표관리자, 부관리자만 해당)

### 2.3. 기업 계정 및 구성원 정책

**기업 등록 정책**

기업이 ADMarket에 등록될 때, TVCF 기업회원 DB에서 정보를 자동으로 가져옵니다. 기존 TVCF 기업회원이 아닌 경우, 신규로 기업 정보를 등록할 수 있습니다.

**구성원 추가 및 승인 정책**

개인이 자신의 프로필에 소속 기업을 등록하면, 해당 기업의 구성원 리스트에 '승인 대기' 상태로 자동 노출됩니다. 기업의 대표관리자 또는 부관리자가 이를 승인하면 정식 구성원이 되며, 동시에 TVCF 구성원으로도 등록됩니다.

**권한 구조 정책**

기업 계정 내에는 다음과 같은 권한 구조가 있습니다:

| 역할 | 권한 | 설명 |
|---|---|---|
| **대표관리자** | 최고 권한 | 기업 정보 관리, 구성원 승인/삭제, 부관리자 지정, 모든 프로젝트 관리 |
| **부관리자** | 관리 권한 | 대표관리자 고유 권한 제외한 모든 권한 (구성원 승인/삭제 가능) |
| **일반직원** | 기본 권한 | 프로젝트 의뢰/참여 기능만 사용 가능 |
| **승인 대기** | 제한 권한 | 승인 전까지 플랫폼 기능 사용 불가 |

**구성원 탈퇴 정책**

구성원이 탈퇴를 신청할 경우, 담당 중인 프로젝트를 다른 구성원에게 이관한 후에만 탈퇴가 가능합니다. 이관 완료 후 대표관리자 또는 부관리자의 승인으로 탈퇴 처리됩니다.

## 3. 프로젝트 정책

### 3.1. 프로젝트 등록 및 승인 정책

**등록 주체**

다음 역할을 가진 회원은 프로젝트를 등록할 수 있습니다:
- 개인 광고주
- 기업 광고주 (승인된 구성원)
- 대행사 (의뢰 역할)
- 컨설턴트

**정보 입력 정책**

프로젝트 등록 시 다음 정보는 필수 입력 항목입니다:
- 프로젝트 제목 및 설명
- 광고 목적 및 제작 기법
- 예산 정보
- 주요 일정 (접수, 마감, 발표, 납품)
- 담당자 정보
- 참여 기업 조건

선택 입력 항목:
- 경쟁사 수행 기업 제외 목록
- 제출 자료 양식
- 추가 설명 및 참고 자료

**임시 저장 정책**

작성 중인 프로젝트는 임시저장할 수 있으며, 최대 30일 동안 보관됩니다. 30일 이후 자동으로 삭제됩니다. 사용자는 임시저장된 프로젝트를 언제든지 불러와 이어서 작성할 수 있습니다.

**승인 절차 정책**

등록된 프로젝트는 다음 단계를 거쳐 승인됩니다:

1. **사용자 제출**: 사용자가 완성된 프로젝트를 운영자에게 승인 요청
2. **운영자 검토**: 운영자가 프로젝트 정보의 진정성, 내용의 적합성, 정책 준수 여부 등을 판단
3. **승인 또는 반려**: 
   - **승인**: 프로젝트가 공개되어 참여사 모집 시작
   - **반려**: 수정 사항을 사용자에게 알리고, 사용자가 수정 후 재신청

**수정 정책**

프로젝트는 운영자 승인 전까지 자유롭게 수정할 수 있습니다. 승인 후에는 다음 항목만 수정 가능합니다:
- 일정 변경
- 담당자 정보 변경
- 추가 설명 및 참고 자료 업로드

프로젝트의 핵심 정보(예산, 참여 조건 등) 변경이 필요한 경우, 운영자에게 변경 요청을 해야 합니다.

### 3.2. 프로젝트 의뢰 방식 정책

**공개 프로젝트 정책**

공개 프로젝트는 모든 참여사가 열람하고 참여 신청할 수 있는 방식입니다.

- **노출 범위**: 모든 회원에게 공개
- **참여 방식**: 자유로운 참여 신청
- **제안서**: 참여사가 자신의 아이디어를 담은 제안서 제출
- **선정 기준**: 의뢰자가 제안서 내용, 참여사의 포트폴리오, 리뷰 등을 종합적으로 검토하여 선정

**비공개 직접의뢰 정책**

비공개 직접의뢰는 의뢰자가 특정 파트너를 지정하여 1:1로 프로젝트를 의뢰하는 방식입니다.

- **노출 범위**: 지정된 파트너만 열람 가능
- **참여 방식**: 초대받은 파트너만 제안 또는 수락/거절 가능
- **제안서**: 필요시 제안서 제출 요청 가능
- **선정 기준**: 의뢰자와 파트너 간의 협의를 통해 선정

**컨설턴트 의뢰 정책**

컨설턴트 의뢰는 ADMarket의 전문가 컨설턴트에게 프로젝트 진행을 위임하는 유료 서비스입니다.

- **서비스 범위**: 파트너 추천, 프로젝트 진행 관리 등
- **수수료**: 별도 계약에 따름
- **선정 기준**: 컨설턴트의 전문적 판단에 따라 추천

### 3.3. 프로젝트 상태 관리 정책

**상태 정의**

모든 프로젝트는 다음과 같은 명확한 상태를 가집니다:

| 단계 | 상태 | 설명 |
|---|---|---|
| **등록** | DRAFT | 작성 중 |
| | REQUESTED | 승인 요청 |
| | APPROVED | 승인 완료 |
| | PROPOSAL_OPEN | 제안 접수 중 |
| | PROPOSAL_CLOSED | 제안 접수 마감 |
| **접수** | OT_SCHEDULED | OT 예정 |
| | OT_COMPLETED | OT 완료 |
| | PT_SCHEDULED | PT 예정 |
| | PT_COMPLETED | PT 완료 |
| | SELECTED | 파트너 선정 |
| **계약** | CONTRACT | 계약 완료 |
| **제작** | SHOOTING | 촬영 중 |
| | EDITING | 후반 작업 |
| | FINAL_APPROVED | 최종 승인 |
| **완료** | COMPLETE | 프로젝트 종료 |
| **중단/취소** | STOPPED | 중단됨 |
| | CANCELLED | 취소됨 |

**상태 전환 정책**

프로젝트의 상태는 다음 규칙에 따라 전환됩니다:
- 각 단계는 순차적으로 진행되어야 합니다.
- 상태 전환 시 관련자에게 자동으로 알림이 전달됩니다.
- 특정 상태에서는 되돌아갈 수 없습니다. (예: 계약 완료 후 접수 단계로 돌아갈 수 없음)

## 4. 참여 및 선정 정책

### 4.1. 참여 자격 정책

**참여 가능 대상**

다음 조건을 만족하는 회원은 프로젝트에 참여할 수 있습니다:
- ADMarket 회원 가입 완료
- 기업 계정의 경우, 대표관리자 또는 부관리자의 승인 완료
- 플랫폼 제재 대상이 아님

**참여 제한**

다음 경우 프로젝트 참여가 제한됩니다:
- 경쟁사 수행 기업으로 지정된 경우
- 의뢰자가 설정한 참여 조건을 충족하지 못하는 경우
- 과거 프로젝트에서 계약 불이행 이력이 있는 경우

### 4.2. 제안서 제출 정책

**제안서 요구사항**

제안서는 다음을 포함해야 합니다:
- 프로젝트에 대한 이해도 및 창의적 아이디어
- 제작 계획 및 일정
- 예상 비용 및 지급 조건
- 참여 팀 소개 및 경험

**제안서 제출 기한**

의뢰자가 설정한 제출 마감일까지 제안서를 제출해야 합니다. 마감일 이후 제출된 제안서는 검토 대상에서 제외될 수 있습니다.

**제안서 관리**

- 제안서는 여러 버전으로 수정 및 재제출할 수 있습니다.
- 최종 제출된 제안서만 의뢰자에게 평가됩니다.
- 제안서는 프로젝트 종료 후 최소 1년간 보관됩니다.

### 4.3. 파트너 선정 정책

**선정 기준**

의뢰자는 다음 기준을 고려하여 파트너를 선정합니다:
- 제안서의 창의성 및 실현 가능성
- 참여사의 포트폴리오 및 경험
- 과거 프로젝트 리뷰 및 평점
- 예산 및 일정 적합성
- 팀의 전문성 및 역량

**선정 결과 공개**

선정 결과는 모든 지원자에게 투명하게 공개됩니다. 선정되지 않은 참여사에게도 그 사실을 알립니다.

**선정 후 절차**

선정된 파트너는 다음 절차를 거칩니다:
1. 계약 정보 작성 및 확인
2. 최종 계약 확정
3. 프로젝트 제작 진행
4. 산출물 제출 및 확정
5. 정산 처리

## 5. 제작 및 완료 정책

### 5.1. 산출물 관리 정책

**산출물 정의**

산출물은 프로젝트 진행 과정에서 발생하는 모든 결과물을 의미합니다:
- 시안 영상
- 최종 영상
- 관련 문서 (스토리보드, 음향 자료 등)

**산출물 업로드 정책**

- 수행사는 계약에 명시된 일정에 따라 산출물을 업로드해야 합니다.
- 여러 버전의 산출물을 업로드할 수 있으며, 각 버전은 자동으로 관리됩니다.
- 파일 형식 및 크기 제한은 플랫폼 기술 정책에 따릅니다.

**버전 관리 정책**

- 각 산출물 버전은 타임스탬프와 함께 저장됩니다.
- 의뢰자는 모든 버전을 확인할 수 있습니다.
- 최종 확정 후에는 이전 버전도 보관됩니다.

### 5.2. 검수 및 확정 정책

**검수 절차**

의뢰자는 제출된 산출물을 다음과 같이 처리할 수 있습니다:
1. **승인**: 산출물이 요구사항을 충족하는 경우
2. **수정 요청**: 수정이 필요한 경우, 구체적인 수정 사항을 명시
3. **거절**: 산출물이 요구사항을 충족하지 못하는 경우

**수정 요청 정책**

- 수정 요청 시 구체적인 사항을 명시해야 합니다.
- 수행사는 합리적인 기간 내에 수정을 완료해야 합니다.
- 수정 횟수는 계약에 명시된 범위를 초과할 수 없습니다.

**최종 확정**

의뢰자가 '확정' 버튼을 클릭하면 산출물이 최종 확정되며, 이후 수정은 불가능합니다.

### 5.3. 정산 정책

**정산 기준**

정산은 계약서에 명시된 조건에 따라 진행됩니다:
- **계약금**: 계약 체결 시 지급 (일반적으로 총액의 30%)
- **중도금**: 제작 진행 중 지급 (일반적으로 총액의 40%)
- **잔금**: 산출물 최종 확정 후 지급 (일반적으로 총액의 30%)

**정산 서류**

수행사는 다음 서류를 제출해야 합니다:
- 세금계산서 또는 영수증
- 지급 계좌 정보
- 기타 필요한 서류

**정산 기한**

정산은 산출물 최종 확정 후 7일 이내에 완료되어야 합니다. 특별한 사유가 있는 경우 양측 합의 하에 연장될 수 있습니다.

### 5.4. 상호 리뷰 정책

**리뷰 작성 의무**

프로젝트가 최종 완료되면, 의뢰사와 선정사는 서로의 업무 수행에 대해 리뷰와 평점을 남겨야 합니다.

**리뷰 내용**

리뷰에는 다음이 포함될 수 있습니다:
- 프로젝트 진행 과정에서의 커뮤니케이션
- 산출물의 품질
- 일정 준수 여부
- 전문성 및 역량
- 기타 의견

**평점 기준**

평점은 1점부터 5점까지 부여할 수 있습니다:
- **5점**: 매우 만족
- **4점**: 만족
- **3점**: 보통
- **2점**: 불만족
- **1점**: 매우 불만족

**리뷰 공개 정책**

리뷰는 작성자의 동의 하에 공개되며, 각 회원의 프로필에 누적됩니다. 리뷰는 향후 다른 프로젝트의 매칭 기준으로 활용됩니다.

## 6. 운영 및 제재 정책

### 6.1. 운영자 역할 정책

**운영자 책임**

운영자는 다음 업무를 수행합니다:
- 프로젝트 승인 및 검수
- 사업자 정보 인증
- 분쟁 조정 및 중재
- 허위 정보 필터링
- 정책 위반 회원 제재
- 플랫폼 안정성 유지

**운영자 중립성**

운영자는 모든 회원을 공평하게 대우하며, 개인적 판단이 아닌 정책에 따라 업무를 수행합니다.

### 6.2. 금지 행위 정책

**금지 행위**

다음 행위는 엄격히 금지됩니다:

| 행위 | 설명 |
|---|---|
| **허위 정보 등록** | 거짓된 기업 정보, 포트폴리오, 경력 등을 등록하는 행위 |
| **무단 정보 수집** | 플랫폼 외부 도구를 사용하여 회원 정보를 수집하는 행위 |
| **비방 및 욕설** | 다른 회원을 비방하거나 욕설을 사용하는 행위 |
| **계약 내용 불이행** | 계약서에 명시된 내용을 이행하지 않는 행위 |
| **부정 거래** | 거짓된 거래를 기록하거나 조작하는 행위 |
| **개인정보 유출** | 다른 회원의 개인정보를 무단으로 공개하는 행위 |
| **스팸 및 광고** | 플랫폼에서 허가하지 않은 광고나 스팸을 발송하는 행위 |

### 6.3. 제재 정책

**제재 단계**

금지 행위가 적발될 경우, 다음과 같은 차등적인 제재가 가해집니다:

| 위반 수위 | 제재 조치 | 기간 |
|---|---|---|
| **경미** | 경고 및 주의 | — |
| **중간** | 서비스 이용 제한 | 1개월 ~ 3개월 |
| **심각** | 계정 정지 | 3개월 ~ 1년 |
| **극심** | 계정 영구 삭제 | 영구 |

**제재 절차**

1. **적발**: 운영자가 금지 행위를 적발
2. **통보**: 해당 회원에게 위반 사실 통보
3. **소명 기회**: 회원이 소명할 수 있는 기회 제공 (7일)
4. **결정**: 운영자가 최종 제재 결정
5. **이의 제기**: 회원이 이의를 제기할 수 있음 (14일)

**제재 해제**

제재 기간이 만료되면 자동으로 해제됩니다. 단, 극심한 위반의 경우 해제 신청을 통해 재검토될 수 있습니다.

## 7. 커뮤니케이션 정책

### 7.1. 메시지 정책

**메시지 사용 목적**

메시지는 프로젝트 관련 커뮤니케이션을 위해서만 사용되어야 합니다. 개인적인 용도나 광고 목적으로 사용할 수 없습니다.

**메시지 보관**

메시지는 프로젝트 종료 후 최소 1년간 보관됩니다. 이는 분쟁 발생 시 증거 자료로 활용될 수 있습니다.

### 7.2. 알림 정책

**알림 발송 기준**

다음 경우에 알림이 자동으로 발송됩니다:
- 프로젝트 상태 변경
- 새로운 메시지 수신
- 참여 신청 접수
- 제안서 제출
- 계약 확정 요청
- 산출물 업로드
- 리뷰 작성 요청

**알림 채널**

알림은 다음 채널을 통해 전달될 수 있습니다:
- 플랫폼 내 알림
- 이메일
- 카카오톡 (선택사항)

사용자는 알림 설정에서 받고 싶은 알림의 종류와 채널을 선택할 수 있습니다.

## 8. 데이터 보관 및 삭제 정책

### 8.1. 데이터 보관 기간

| 데이터 유형 | 보관 기간 | 사유 |
|---|---|---|
| **프로젝트 정보** | 5년 | 세무 및 법적 증거 |
| **계약서 및 정산 자료** | 5년 | 세무 및 법적 증거 |
| **메시지** | 1년 | 분쟁 해결 증거 |
| **리뷰 및 평점** | 무제한 | 신뢰도 데이터 |
| **개인정보** | 탈퇴 시 삭제 | 개인정보보호법 준수 |

### 8.2. 데이터 삭제 정책

회원이 탈퇴할 경우, 다음과 같이 데이터가 처리됩니다:
- **개인정보**: 즉시 삭제
- **프로젝트 정보**: 법정 보존 기간 동안 보관 후 삭제
- **메시지**: 1년 보관 후 삭제
- **리뷰**: 유지 (익명 처리 가능)

## 9. 분쟁 처리 정책

### 9.1. 분쟁 해결 절차

**분쟁 신고**

분쟁이 발생한 경우, 당사자는 플랫폼에 신고할 수 있습니다. 신고 시 다음 정보를 제공해야 합니다:
- 프로젝트 ID
- 분쟁의 구체적인 내용
- 관련 증거 자료 (메시지, 계약서 등)

**운영자 중재**

운영자는 신고 접수 후 7일 이내에 양측 의견을 청취하고 중재합니다. 필요시 전문가 자문을 구할 수 있습니다.

**분쟁 해결**

중재 결과에 따라 다음과 같은 조치가 취해질 수 있습니다:
- 합의 권유
- 손해배상 권고
- 계약 해제 권고
- 법적 절차 안내
